# Requirements: client-main-modularize

## 背景

- `src/client/main.ts` が大きく、通信・状態管理・描画・UIが1ファイルに混在している。
- 仕様変更時の影響範囲が読みにくく、局所修正の安全性が下がっている。

## 要求

1. `main.ts` の責務を `network/state/renderer/ui` に分割する。
2. `main.ts` は起動・モジュール接続・サーバーメッセージ分配を中心とした薄い構成にする。
3. 既存挙動（観戦切替、AIテスト用ロビー設定、補間描画、再接続）を維持する。
4. `check/build/test` を通過させる。
5. 分割方針を docs に記録する。

## 非要求

- UI/ゲームデザインそのものの刷新は対象外。
- 通信プロトコル変更は対象外。
